## 第五部分：协作沟通模块详细设计

---

## 1. 模块概述

协作沟通模块负责实现员工本人、分身智能体之间的实时对话、协作请求、进度同步等功能，是平台实现"智能协作"的核心模块。

### 1.1 模块职责

| 子模块 | 职责描述 | 对应服务 |
|--------|----------|----------|
| 即时消息 | 实时对话、群聊、消息推送 | message-service |
| 协作请求 | 协作发起、响应、进度同步 | collaboration-service |
| 技能匹配 | 技能搜索、智能推荐 | avatar-service |
| 协作评价 | 协作完成评价、评分统计 | collaboration-service |

## 2. 即时消息功能

### 2.1 功能描述

支持员工与员工、员工与分身、分身与分身之间的实时对话，支持单聊和群聊，支持多种消息类型。

### 2.2 消息类型定义

| 消息类型 | 类型码 | 描述 | 支持操作 |
|----------|--------|------|----------|
| 文字消息 | 1 | 纯文本内容，支持表情 | 撤回、编辑、复制、转发 |
| 图片消息 | 2 | 图片文件，支持预览 | 撤回、下载、转发 |
| 文件消息 | 3 | 各类文件附件 | 撤回、下载、转发 |
| 语音消息 | 4 | 语音录制，支持播放 | 撤回、转文字 |
| 链接消息 | 5 | URL链接，支持预览卡片 | 撤回、打开、转发 |
| 卡片消息 | 6 | 结构化卡片，如任务卡、审批卡 | 撤回、操作 |
| 系统消息 | 7 | 系统通知，如成员变更 | 无 |

### 2.5 接口设计

**接口1：创建会话**

```yaml
接口路径: POST /api/v1/conversations
请求头:
  Authorization: Bearer {token}
  Content-Type: application/json
请求参数:
  conversation_type: integer # 会话类型：1-单聊 2-群聊
  members: array             # 成员列表
    - member_type: integer   # 成员类型：1-用户 2-分身
      member_id: string      # 成员ID
  conversation_name: string  # 会话名称（群聊必填）
  conversation_avatar: string # 会话头像（群聊选填）
响应参数:
  code: integer
  message: string
  data:
    conversation_id: string
    conversation_type: integer
    conversation_name: string
    conversation_avatar: string
    member_count: integer
    created_time: string
业务规则:
  1. 单聊会话自动查找已存在的会话
  2. 群聊至少需要3个成员
  3. 群聊最多500人
  4. 自动将创建者设为群主
```

**接口5：发送消息**

```yaml
接口路径: POST /api/v1/conversations/{conversation_id}/messages
请求头:
  Authorization: Bearer {token}
  Content-Type: application/json
请求参数:
  message_type: integer      # 消息类型，必填
  content: string            # 文字内容，文字消息必填
  media_url: string          # 媒体文件URL，文件类消息必填
  media_name: string         # 媒体文件名
  media_size: integer        # 媒体文件大小
  media_duration: integer    # 媒体时长（音视频）
  thumbnail_url: string      # 缩略图URL
  extra_data: object         # 扩展数据
  reply_message_id: string   # 回复的消息ID
  at_user_ids: array         # @的用户ID列表
  is_at_all: boolean         # 是否@所有人
响应参数:
  code: integer
  message: string
  data:
    message_id: string
    created_time: string
业务规则:
  1. 消息内容需过滤敏感词
  2. 媒体文件需先通过文件服务上传
  3. @消息需通知被@的用户
  4. 群主/管理员可@所有人
  5. 消息发送后通过WebSocket推送
```

**接口6：撤回消息**

```yaml
接口路径: POST /api/v1/messages/{message_id}/revoke
请求头:
  Authorization: Bearer {token}
响应参数:
  code: integer
  message: string
  data:
    revoked: boolean
业务规则:
  1. 只能撤回自己发送的消息
  2. 撤回时间限制：10分钟内
  3. 群主/管理员可撤回任意成员消息
  4. 撤回后通知所有会话成员
```

**接口7：编辑消息**

## 3. 协作请求功能

### 3.1 功能描述

员工可以通过分身向其他员工/分身发起协作请求，接收方可以接受、拒绝或延期，双方可以同步协作进度。

### 3.4 接口设计

**接口1：发起协作请求**

```yaml
接口路径: POST /api/v1/collaborations
请求头:
  Authorization: Bearer {token}
  Content-Type: application/json
请求参数:
  title: string              # 协作标题，必填，5-200字符
  description: string        # 协作描述，选填
  receiver_type: integer     # 接收方类型：1-用户 2-分身
  receiver_id: string        # 接收方ID，必填
  priority: integer          # 优先级：1-紧急 2-高 3-普通 4-低
  deadline: string           # 截止时间，必填，格式YYYY-MM-DD HH:mm:ss
  attachments: array         # 附件列表
    - file_name: string
      file_url: string
      file_size: integer
      file_type: string
响应参数:
  code: integer
  message: string
  data:
    collab_id: string
    collab_code: string
    status: integer
    created_time: string
业务规则:
  1. 发起人类型根据当前操作主体判断（用户/分身）
  2. 协作编号自动生成：COLLAB+日期+序号
  3. 截止时间不能早于当前时间
```

## 6. 消息与协作通知

### 6.1 通知触发规则

| 事件 | 通知对象 | 通知方式 | 通知内容 |
|------|----------|----------|----------|
| 新消息 | 接收方 | 站内+推送 | "{发送人}给你发送了一条消息" |
| @提醒 | 被@的人 | 站内+推送 | "{发送人}在{会话}中@了你" |
| 协作请求 | 接收方 | 站内+推送+邮件 | "你收到一条新的协作请求" |
| 协作接受 | 发起方 | 站内+推送 | "{接收方}接受了你的协作请求" |
| 协作拒绝 | 发起方 | 站内+推送 | "{接收方}拒绝了你的协作请求" |
| 进度更新 | 发起方 | 站内 | "协作进度已更新至{进度}%" |
| 协作完成 | 发起方 | 站内+推送 | "{接收方}完成了协作" |
| 协作超时 | 双方 | 站内+推送 | "协作即将/已经超时" |

### 6.2 消息推送策略

```yaml
推送策略:
  在线用户:
    - 通过WebSocket实时推送
    - 更新会话未读数
    
  离线用户:
    - 写入消息队列
    - 通过APNs/FCM推送
    - 聚合推送（避免频繁打扰）
    
  免打扰用户:
    - 不发送推送
    - 保留未读状态
    - 免打扰结束后批量推送
    
消息聚合规则:
  - 同一会话5分钟内消息聚合
  - 聚合消息显示"你有N条新消息"
  - 协作相关通知不聚合，单独推送
```
