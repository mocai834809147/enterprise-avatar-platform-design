# 企业伴身智能平台 - 详细设计文档

## 第六部分：流程管理模块详细设计

---

## 1. 模块概述

流程管理模块为员工提供个人任务管理和工作流程审批功能，实现任务的创建、提醒、进度管理，以及各类审批流程的配置、发起、处理和归档，包含个人任务管理、工作流程审批、流程模板管理三大子模块。

---

## 2. 个人任务管理

### 2.1 功能清单

| 功能编号 | 功能名称 | 功能描述 | 优先级 |
|----------|----------|----------|--------|
| TASK-001 | 创建任务 | 创建个人任务 | P0 |
| TASK-002 | 任务编辑 | 编辑任务信息 | P0 |
| TASK-003 | 任务提醒 | 设置任务提醒 | P0 |
| TASK-004 | 状态管理 | 任务状态切换 | P0 |
| TASK-005 | 进度更新 | 更新任务进度 | P0 |
| TASK-006 | 任务筛选 | 按条件筛选任务 | P1 |
| TASK-007 | 子任务 | 创建子任务 | P1 |
| TASK-008 | 任务分享 | 分享任务给协作人 | P2 |

### 2.2 任务状态

| 状态值 | 状态名称 | 颜色标识 | 说明 |
|--------|----------|----------|------|
| 1 | 待处理 | 灰色 | 任务已创建，未开始 |
| 2 | 进行中 | 蓝色 | 任务正在处理中 |
| 3 | 已完成 | 绿色 | 任务已完成 |
| 4 | 已延期 | 橙色 | 任务已延期 |
| 5 | 已取消 | 红色 | 任务已取消 |

### 2.3 创建任务

```json
// 请求
POST /api/v1/tasks
{
  "title": "完成用户模块API设计",
  "description": "设计用户注册、登录、信息管理等接口",
  "priority": 3,  // 1-低, 2-中, 3-高, 4-紧急
  "start_date": "2024-01-15",
  "due_date": "2024-01-20",
  "due_time": "18:00:00",
  "assignee_id": null,  // 负责人，默认自己
  "related_users": [10002, 10003],  // 相关人员
  "tags": ["开发", "API"],
  "attachments": [
    {
      "file_id": 80001,
      "file_name": "需求文档.docx"
    }
  ],
  "reminder_settings": {
    "enabled": true,
    "reminders": [
      {"type": "before_due", "time_value": 1, "time_unit": "hour"},
      {"type": "before_due", "time_value": 1, "time_unit": "day"}
    ]
  },
  "parent_task_id": null,  // 父任务ID，创建子任务时使用
  "via_avatar": true  // 是否通过分身创建
}

// 响应
{
  "code": 200,
  "data": {
    "task_id": 100001,
    "task_code": "TASK_20240115_001",
    "title": "完成用户模块API设计",
    "status": 1,
    "priority": 3,
    "due_date": "2024-01-20",
    "created_at": "2024-01-15 09:00:00"
  }
}
```

## 3. 工作流程审批

### 3.1 功能清单

| 功能编号 | 功能名称 | 功能描述 | 优先级 |
|----------|----------|----------|--------|
| APPR-001 | 审批流程配置 | 管理员配置审批流程 | P0 |
| APPR-002 | 发起审批 | 员工发起审批申请 | P0 |
| APPR-003 | 审批处理 | 审批人处理审批 | P0 |
| APPR-004 | 进度跟踪 | 跟踪审批进度 | P0 |
| APPR-005 | 审批撤回 | 撤回已提交的审批 | P0 |

| 状态值 | 状态名称 | 说明 |

### 5.2 审批人解析规则

| 审批人类型 | 解析规则 |
|------------|----------|
| user | 直接使用配置的用户ID |
| role | 查找具有指定角色的用户 |
| department_leader | 获取申请人部门负责人 |
| direct_leader | 获取申请人直属上级 |
| form_field | 从表单字段获取用户ID |
| self | 申请人自己（用于确认节点） |

### 5.3 条件表达式

```json
{
  "conditions": [
    {
      "condition_id": "cond_1",

      },
      "target_node": "node_3",
      "else_node": "node_2"

// in - 包含于
```
