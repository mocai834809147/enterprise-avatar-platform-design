# 企业伴身智能平台 - 详细设计文档

## 第四部分：分身智能体模块详细设计

---

## 1. 模块概述

分身智能体模块是平台的核心模块，负责员工分身的创建、配置、智能响应、知识库管理、记忆能力等功能，是实现"数字化分身"的关键。

### 1.1 模块职责

| 子模块 | 职责描述 | 对应服务 |
|--------|----------|----------|
| 分身管理 | 分身创建、配置、状态管理 | avatar-service |
| 技能管理 | 技能标签CRUD、技能搜索匹配 | avatar-service |
| 响应设置 | 自动回复、响应规则、智能转接 | avatar-service |
| 知识库 | 知识库CRUD、内容管理、调用规则 | knowledge-service |
| 记忆能力 | 记忆管理、规则配置、纠错优化 | avatar-service |
| AI智能 | 意图识别、智能问答、内容推荐 | ai-service |

## 2. 分身创建与初始化

### 2.1 功能描述

员工登录后可一键创建专属分身智能体，系统自动分配唯一ID，支持自定义分身名称、头像、形象等。

### 2.3 接口设计

**接口1：创建分身**

```yaml
接口路径: POST /api/v1/avatars
请求头:
  Authorization: Bearer {token}
  Content-Type: application/json
请求参数:
  avatar_name: string        # 分身名称，选填，默认与员工姓名一致
  avatar_image: string       # 分身头像URL，选填，默认使用用户头像
  avatar_type: integer       # 形象类型：1-真人头像 2-虚拟形象 3-卡通形象
  signature: string          # 个性签名，选填
响应参数:
  code: integer
  message: string
  data:
    avatar_id: string        # 分身ID
    avatar_code: string      # 分身编码
    avatar_name: string      # 分身名称
    avatar_image: string     # 分身头像URL
    status: integer          # 状态：1-在线
    created_time: string     # 创建时间
业务规则:
  1. 每个用户只能创建一个分身
  2. 分身编码自动生成：AVT+企业编码后4位+时间戳+随机数
  3. 创建时自动同步用户基础信息到分身
  4. 自动创建"常见问题"默认知识库
  5. 自动生成默认响应规则
  6. 分身创建后状态默认为"在线"
```

**接口2：获取分身信息**

```yaml
接口路径: GET /api/v1/avatars/{avatar_id}
请求头:
  Authorization: Bearer {token}
响应参数:
  code: integer
  message: string
  data:
    id: string               # 分身ID
    avatar_code: string      # 分身编码
    avatar_name: string      # 分身名称
    avatar_image: string     # 分身头像
    avatar_type: integer     # 形象类型
    virtual_model_id: string # 虚拟形象模型ID
    signature: string        # 个性签名
    status: integer          # 状态：1-在线 2-忙碌 3-离开 4-休假 5-离线
    status_message: string   # 状态说明
    auto_reply_enabled: boolean  # 自动回复开关
    offline_reply_content: string # 离线自动回复
    ai_enabled: boolean      # AI智能响应开关
    response_delay: integer  # 响应延迟(秒)
    user_info:               # 关联用户信息
      user_id: string
      real_name: string
      dept_name: string
      position_name: string
      phone: string          # 脱敏
      email: string
    skill_count: integer     # 技能标签数
    kb_count: integer        # 知识库数
    memory_count: integer    # 记忆条数
    statistics:              # 统计数据
      dialogue_count: integer
      response_rate: number
      avg_response_time: integer
    created_time: string
```

**接口3：更新分身基础信息**

```yaml
接口路径: PUT /api/v1/avatars/{avatar_id}
请求头:
  Authorization: Bearer {token}
  Content-Type: application/json
请求参数:
  avatar_name: string        # 分身名称
  avatar_image: string       # 分身头像URL
  avatar_type: integer       # 形象类型
  virtual_model_id: string   # 虚拟形象模型ID
  signature: string          # 个性签名
响应参数:
  code: integer
  message: string
  data:
    updated: boolean
业务规则:
  1. 只能修改自己的分身
  2. 名称长度2-50字符
  3. 头像需通过文件服务上传
```

**接口4：设置分身状态**

```yaml
接口路径: PUT /api/v1/avatars/{avatar_id}/status
请求头:
  Authorization: Bearer {token}
  Content-Type: application/json
请求参数:
  status: integer            # 状态：1-在线 2-忙碌 3-离开 4-休假 5-离线
  status_message: string     # 状态说明，选填
  auto_offline_time: string  # 自动离线时间，选填，格式HH:mm
  auto_online_time: string   # 自动上线时间，选填，格式HH:mm
响应参数:
  code: integer
  message: string
  data:
    updated: boolean
    old_status: integer
    new_status: integer
业务规则:
  1. 状态变更记录日志
  2. 状态变更实时推送给所有关联方
  3. 设置休假状态时启用离线自动回复
  4. 支持定时自动切换状态
```

## 3. 分身技能管理

### 3.1 功能描述

员工可以添加、编辑、删除分身的技能标签，设置技能熟练度，支持按技能标签搜索其他员工分身。

### 3.2 接口设计

**接口1：获取技能列表**

```yaml
接口路径: GET /api/v1/avatars/{avatar_id}/skills
请求头:
  Authorization: Bearer {token}
请求参数:
  category: string           # 技能分类筛选
  is_public: boolean         # 是否公开筛选
响应参数:
  code: integer
  message: string
  data:
    - id: string
      skill_name: string
      skill_category: string
      proficiency: integer   # 熟练度：1-初级 2-中级 3-高级 4-专家
      proficiency_name: string
      description: string
      is_public: boolean
      verify_status: integer # 认证状态
      sort_order: integer
      created_time: string
```

**接口2：添加技能**

```yaml
接口路径: POST /api/v1/avatars/{avatar_id}/skills
请求头:
  Authorization: Bearer {token}
  Content-Type: application/json
请求参数:
  skill_name: string         # 技能名称，必填，2-100字符
  skill_category: string     # 技能分类，选填
  proficiency: integer       # 熟练度，必填，1-4
  description: string        # 技能描述，选填
  is_public: boolean         # 是否公开，默认true
  sort_order: integer        # 排序序号，选填
响应参数:
  code: integer
  message: string
  data:
    id: string
    skill_name: string
业务规则:
  1. 同一分身技能名称不可重复
  2. 技能数量限制：单个分身最多50个技能
  3. 添加技能自动同步到ES索引
  4. 公开技能可被其他员工搜索到
```

**接口3：更新技能**

