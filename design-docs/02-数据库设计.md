# 企业伴身智能平台 - 详细设计文档

## 第二部分：数据库设计与数据模型

---

## 1. 数据库设计原则

### 1.1 设计规范

| 规范项 | 规范说明 |
|--------|----------|
| 命名规范 | 表名：小写字母+下划线，如 `t_user`、`t_avatar`；字段名：小写字母+下划线 |
| 主键设计 | 使用雪花算法生成的分布式ID（BIGINT），字段名统一为 `id` |
| 字符集 | 统一使用 `utf8mb4`，排序规则 `utf8mb4_general_ci` |
| 时间字段 | 统一使用 `DATETIME` 类型，时区为东八区 |
| 软删除 | 所有业务表增加 `is_deleted` 字段（TINYINT，0-未删除，1-已删除） |
| 审计字段 | 所有表必须包含：`created_by`、`created_time`、`updated_by`、`updated_time` |
| 索引规范 | 单表索引不超过6个，联合索引字段不超过5个 |
| 字段长度 | VARCHAR长度：姓名50、标题200、描述500、内容TEXT |

### 1.2 分库分表策略

| 数据类型 | 分库策略 | 分表策略 | 说明 |
|----------|----------|----------|------|
| 用户数据 | 按企业ID取模 | 单表 | 企业维度隔离 |
| 消息数据 | 按企业ID取模 | 按月分表 | 数据量大，按时间归档 |
| 任务数据 | 按企业ID取模 | 按月分表 | 历史数据归档 |
| 审批数据 | 按企业ID取模 | 按月分表 | 历史数据归档 |
| 日志数据 | 单库 | 按日分表 | 日志数据量大 |
| 统计数据 | 单库 | 按类型分表 | 聚合数据 |

## 4. 索引设计规范

### 4.1 索引命名规范

| 索引类型 | 命名格式 | 示例 |
|----------|----------|------|
| 主键 | pk_{表名} 或直接 PRIMARY | PRIMARY KEY |
| 唯一索引 | uk_{字段名} | uk_user_code |
| 普通索引 | idx_{字段名} | idx_corp_id |
| 联合索引 | idx_{字段1}_{字段2} | idx_corp_status |
| 全文索引 | ft_{字段名} | ft_content |
