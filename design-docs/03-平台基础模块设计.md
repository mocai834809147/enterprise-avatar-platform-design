# 企业伴身智能平台 - 详细设计文档

## 第三部分：平台基础模块详细设计

---

## 1. 模块概述

平台基础模块是整个系统的底层支撑，包含企业注册与配置、员工账号管理、认证授权、消息通知、多端同步等核心功能，为其他业务模块提供基础能力支持。

### 1.1 模块职责

| 子模块 | 职责描述 | 对应服务 |
|--------|----------|----------|
| 企业管理 | 企业注册、认证、配置、部门岗位管理 | corp-service |
| 用户管理 | 员工账号CRUD、权限分配、个人信息管理 | user-service |
| 认证授权 | 登录认证、Token管理、权限校验 | auth-service |
| 消息通知 | 站内通知、短信、邮件、推送 | notification-service |
| 文件服务 | 文件上传下载、存储管理 | file-service |

#### 2.1.3 接口设计

**接口1：企业注册提交**

```yaml
接口路径: POST /api/v1/corp/register
请求头:
  Content-Type: application/json
请求参数:
  corp_name: string          # 企业名称，必填，2-100字符
  corp_short_name: string    # 企业简称，选填，2-50字符
  contact_name: string       # 联系人姓名，必填，2-50字符
  contact_phone: string      # 联系人手机，必填，11位手机号
  contact_email: string      # 联系人邮箱，必填，邮箱格式
  phone_code: string         # 手机验证码，必填，6位数字
  email_code: string         # 邮箱验证码，必填，6位数字
  admin_username: string     # 管理员账号，必填，4-20字符，字母数字
  admin_password: string     # 管理员密码，必填，8-20字符，含字母数字
  industry: string           # 所属行业，选填
  scale: integer             # 企业规模，选填，1-5
  province: string           # 省份，选填
  city: string               # 城市，选填
  district: string           # 区县，选填
  address: string            # 详细地址，选填
响应参数:
  code: integer              # 响应码，200成功
  message: string            # 响应信息
  data:
    corp_id: string          # 企业ID
    corp_code: string        # 企业编码
    admin_user_id: string    # 管理员用户ID
业务规则:
  1. 企业名称全局唯一，不可重复
  2. 联系人手机号需通过验证码校验
  3. 联系人邮箱需通过验证码校验
  4. 管理员账号在企业内唯一
  5. 密码需满足复杂度要求：至少包含字母和数字
  6. 注册成功后自动创建企业和管理员账号
  7. 默认开通100人试用额度，有效期30天
```

**接口2：企业认证提交**

```yaml
接口路径: POST /api/v1/corp/certification
请求头:
  Authorization: Bearer {token}
  Content-Type: multipart/form-data
请求参数:
  license_no: string         # 营业执照号，必填，15或18位
  license_image: file        # 营业执照图片，必填，jpg/png，≤5MB
  legal_person_name: string  # 法人姓名，必填
  legal_person_id_no: string # 法人身份证号，必填，18位
  legal_person_id_front: file # 法人身份证正面，必填
  legal_person_id_back: file  # 法人身份证反面，必填
响应参数:
  code: integer
  message: string
  data:
    cert_id: string          # 认证申请ID
    cert_status: integer     # 认证状态：1-认证中
业务规则:
  1. 营业执照号需通过工商接口校验
  2. 法人身份证需进行OCR识别校验
  3. 图片需进行清晰度检测
  4. 提交后进入人工审核流程
  5. 审核结果通过短信和站内信通知
```

#### 2.1.4 数据校验规则

| 字段 | 校验规则 | 错误提示 |
|------|----------|----------|
| corp_name | 必填，2-100字符，不含特殊字符 | 企业名称格式不正确 |
| contact_phone | 必填，11位数字，以1开头 | 手机号格式不正确 |
| contact_email | 必填，标准邮箱格式 | 邮箱格式不正确 |
| admin_username | 必填，4-20字符，字母数字下划线 | 账号格式不正确 |
| admin_password | 必填，8-20字符，含字母和数字 | 密码不满足复杂度要求 |
| license_no | 15或18位统一社会信用代码 | 营业执照号格式不正确 |
| legal_person_id_no | 18位身份证号，校验位正确 | 身份证号格式不正确 |

### 2.2 部门管理功能

#### 2.2.1 功能描述

企业管理员可以创建、编辑、删除部门，设置部门层级结构，指定部门负责人。

#### 2.2.2 接口设计

**接口1：获取部门树**

```yaml
接口路径: GET /api/v1/corp/departments/tree
请求头:
  Authorization: Bearer {token}
请求参数:
  corp_id: string            # 企业ID，必填（从token中获取）
  include_disabled: boolean  # 是否包含禁用部门，默认false
响应参数:
  code: integer
  message: string
  data:
    - id: string             # 部门ID
      dept_code: string      # 部门编码
      dept_name: string      # 部门名称
      parent_id: string      # 父部门ID
      dept_level: integer    # 部门层级
      leader_id: string      # 负责人ID
      leader_name: string    # 负责人姓名
      member_count: integer  # 成员数量
      status: integer        # 状态：0-禁用 1-启用
      sort_order: integer    # 排序序号
      children: array        # 子部门列表（递归结构）
```

**接口2：创建部门**

```yaml
接口路径: POST /api/v1/corp/departments
请求头:
  Authorization: Bearer {token}
  Content-Type: application/json
请求参数:
  dept_name: string          # 部门名称，必填，2-100字符
  dept_code: string          # 部门编码，选填，自动生成
  parent_id: string          # 父部门ID，选填，空为顶级部门
  leader_id: string          # 负责人ID，选填
  description: string        # 部门描述，选填，≤500字符
  sort_order: integer        # 排序序号，选填，默认0
响应参数:
  code: integer
  message: string
  data:
    id: string               # 部门ID
    dept_code: string        # 部门编码
    dept_path: string        # 部门路径
业务规则:
  1. 同一父部门下部门名称不可重复
  2. 部门层级最多支持5级
  3. 负责人必须是企业内有效员工
  4. 部门编码自动生成规则：DEPT+时间戳+随机数
  5. 创建成功后自动计算部门路径
```

**接口3：更新部门**

```yaml
接口路径: PUT /api/v1/corp/departments/{dept_id}
请求头:
  Authorization: Bearer {token}
  Content-Type: application/json
请求参数:
  dept_name: string          # 部门名称，选填
  parent_id: string          # 父部门ID，选填
  leader_id: string          # 负责人ID，选填
  description: string        # 部门描述，选填
  sort_order: integer        # 排序序号，选填
  status: integer            # 状态，选填
响应参数:
  code: integer
  message: string
  data:
    updated: boolean         # 是否更新成功
业务规则:
  1. 不能将部门移动到自己的子部门下
  2. 修改父部门时需重新计算部门路径
  3. 禁用部门时，其下所有子部门同时禁用
  4. 禁用部门不影响部门内员工状态
```

**接口4：删除部门**

```yaml
接口路径: DELETE /api/v1/corp/departments/{dept_id}
请求头:
  Authorization: Bearer {token}
请求参数:
  force: boolean             # 是否强制删除，默认false
  transfer_dept_id: string   # 人员转移目标部门，强制删除时必填
响应参数:
  code: integer
  message: string
  data:
    deleted: boolean         # 是否删除成功
    transferred_count: integer # 转移人员数量
业务规则:
  1. 有子部门时不可直接删除
  2. 有员工时需指定转移部门或强制删除
  3. 强制删除时，员工将转移到指定部门
  4. 删除操作为软删除，数据保留
  5. 删除部门记录到操作日志
```

#### 3.1.3 接口设计

**接口1：添加员工**

```yaml
接口路径: POST /api/v1/users
请求头:
  Authorization: Bearer {token}
  Content-Type: application/json
请求参数:
  real_name: string          # 真实姓名，必填，2-50字符
  user_code: string          # 工号，选填，自动生成
  phone: string              # 手机号，必填，11位
  email: string              # 邮箱，选填
  gender: integer            # 性别，选填，0-未知 1-男 2-女
  dept_id: string            # 部门ID，必填
  position_id: string        # 岗位ID，选填
  direct_leader_id: string   # 直属上级ID，选填
  entry_date: string         # 入职日期，选填，格式YYYY-MM-DD
  user_type: integer         # 用户类型，选填，默认1-普通员工
  role_ids: array[string]    # 角色ID列表，选填
响应参数:
  code: integer
  message: string
  data:
    user_id: string          # 用户ID
    user_code: string        # 工号
    username: string         # 登录账号（自动生成）
    temp_password: string    # 临时密码
业务规则:
  1. 手机号在企业内唯一
  2. 工号在企业内唯一，格式：EMP+年份+4位序号
  3. 登录账号自动生成，规则：姓名拼音+工号后4位
  4. 临时密码随机生成，8位字母数字
  5. 添加成功后发送激活邮件/短信
  6. 员工首次登录需修改密码
```

**接口2：批量导入员工**

```yaml
接口路径: POST /api/v1/users/import
请求头:
  Authorization: Bearer {token}
  Content-Type: multipart/form-data
请求参数:
  file: file                 # Excel文件，必填，xlsx格式，≤10MB
  update_exist: boolean      # 是否更新已存在数据，默认false
响应参数:
  code: integer
  message: string
  data:
    total: integer           # 总行数
    success: integer         # 成功数
    fail: integer            # 失败数
    fail_details: array      # 失败详情
      - row: integer         # 行号
        reason: string       # 失败原因
Excel模板字段:
  真实姓名(必填) | 手机号(必填) | 邮箱 | 性别 | 部门名称(必填) | 岗位名称 | 直属上级工号 | 入职日期
业务规则:
  1. Excel文件需使用指定模板
  2. 单次导入最多1000条
  3. 部门名称需与系统中完全匹配
  4. 导入采用事务处理，全部成功才提交
  5. 返回导入结果，包含失败明细
```

## 5. 消息通知模块

### 5.1 功能描述

提供多渠道的消息通知能力，包括站内通知、短信、邮件、移动端推送，支持通知偏好设置和免打扰功能。

### 5.2 通知类型

| 通知类型 | 类型码 | 触发场景 | 默认渠道 |
|----------|--------|----------|----------|

| 审批通知 | APPROVAL | 审批待办、审批结果、催办 | 站内+推送+邮件 |
| 消息通知 | MESSAGE | 新消息、@提醒、群公告 | 站内+推送 |

## 6. 文件服务模块

### 6.1 功能描述

提供统一的文件上传、下载、预览服务，支持图片、文档、音视频等多种文件类型，支持文件安全检测和访问控制。

### 6.2 文件类型限制

| 文件类型 | 允许扩展名 | 最大大小 | 用途场景 |
|----------|------------|----------|----------|
| 图片 | jpg,jpeg,png,gif,webp | 10MB | 头像、附件、知识库 |
| 文档 | doc,docx,xls,xlsx,pdf,ppt,pptx | 50MB | 知识库、附件 |
| 音频 | mp3,wav,m4a | 100MB | 知识库、语音消息 |
| 视频 | mp4,mov,avi | 500MB | 知识库 |
| 压缩包 | zip,rar,7z | 200MB | 批量上传 |

### 6.3 接口设计

**接口1：获取上传凭证**

```yaml
接口路径: POST /api/v1/files/upload-token
请求头:
  Authorization: Bearer {token}
  Content-Type: application/json
请求参数:
  file_name: string          # 文件名，必填
  file_size: integer         # 文件大小(字节)，必填
  content_type: string       # MIME类型，必填
  scene: string              # 上传场景：avatar/attachment/knowledge
  directory: string          # 存储目录，选填
响应参数:
  code: integer
  message: string
  data:
    upload_id: string        # 上传ID
    upload_url: string       # 上传地址
    upload_method: string    # 上传方法：PUT
    headers: object          # 上传请求头
    expire_time: string      # 凭证过期时间
业务规则:
  1. 校验文件类型和大小限制
  2. 根据场景设置存储目录
  3. 凭证有效期30分钟
  4. 支持断点续传
```

**接口2：确认上传完成**

```yaml
接口路径: POST /api/v1/files/upload-complete
请求头:
  Authorization: Bearer {token}
  Content-Type: application/json
请求参数:
  upload_id: string          # 上传ID，必填
  file_key: string           # 文件key，必填
响应参数:
  code: integer
  message: string
  data:
    file_id: string          # 文件ID
    file_url: string         # 文件访问URL
    file_name: string        # 文件名
    file_size: integer       # 文件大小
    content_type: string     # MIME类型
    thumbnail_url: string    # 缩略图URL（图片/视频）
业务规则:
  1. 校验文件是否上传成功
  2. 图片自动生成缩略图
  3. 视频自动截取首帧作为封面
  4. 文档进行病毒扫描
  5. 返回CDN加速URL
```

**接口3：获取下载链接**

```yaml
接口路径: GET /api/v1/files/{file_id}/download-url
请求头:
  Authorization: Bearer {token}

业务规则:

  3. 记录下载日志
```

**接口4：删除文件**

```yaml

请求头:
  Authorization: Bearer {token}
响应参数:

    deleted: boolean
```

### 7.3 WebSocket消息协议

```json
// 消息格式
{
  "type": "string",        // 消息类型
  "seq": "integer",        // 消息序号
  "timestamp": "integer",  // 时间戳(毫秒)
  "data": {}               // 消息数据
}

// 消息类型定义
{
  "CONNECT": "连接建立",
  "HEARTBEAT": "心跳",

  "STATUS_CHANGE": "状态变更",

  "SYNC_RESPONSE": "同步响应",
```

### 7.4 离线同步策略

| 数据类型 | 离线缓存 | 同步策略 | 冲突处理 |
|----------|----------|----------|----------|
| 消息 | 最近1000条 | 增量同步 | 以服务端为准 |
| 任务 | 所有待办 | 全量+增量 | 时间戳对比 |
| 分身配置 | 完整配置 | 全量同步 | 以最新为准 |
| 通知 | 最近100条 | 增量同步 | 以服务端为准 |
| 用户状态 | 实时 | 实时推送 | 以服务端为准 |
